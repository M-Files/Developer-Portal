Blocking
Indicates whether the interface should be blocked while the command is being executed. Useful for longer running commands where further user interaction with the interface could negatively effect user experience or execution, or simply a way to indicate to the user that something is happening, and when it is over.
ConfirmMessage
If specified, the message will be displayed to the user in a confirmation dialog after they've triggered the command, but before its execution. They will have the option to proceed or cancel the command.
DisplayName
The display name to use for the command. Optional.
ExposeErrors
Indicates whether errors should be exposed to the caller in JavaScript. - When true, any errors are not handled by the default handler... Instead they are allowed to bubble up through the promise chain. This allows them to be handled by non-default implementations. - When false, the default handler is used. This will show the message in a MFClientScript.IDashboard.ShowMessage() dialog.
